Polimorfismo

1 / 12

¿Qué es el Polimorfismo?

Del griego "muchas formas". Es la capacidad de objetos de diferentes clases de responder al mismo mensaje (llamada de método) de maneras distintas.

2 / 12

Ejemplo con Herencia

Es la forma clásica de ver el polimorfismo. Diferentes clases hijas implementan el mismo método de la clase padre.

3 / 12

Paso 1: La Clase Padre

Creamos una clase `Animal` con un método `hablar()`.

class Animal:
    def hablar(self):
        # Este método será sobrescrito
        raise NotImplementedError("La subclase debe implementar este método")
4 / 12

Paso 2: Las Clases Hijas

Cada clase hija implementa `hablar()` a su manera.

class Perro(Animal):
    def hablar(self):
        return "¡Guau!"

class Gato(Animal):
    def hablar(self):
        return "¡Miau!"
5 / 12

El Polimorfismo en Acción

Podemos tratar a todos los objetos como `Animal` y llamar al mismo método, obteniendo diferentes comportamientos.

animales = [Perro(), Gato()]

for animal in animales:
    # No necesitamos saber si es un Perro o un Gato
    print(animal.hablar())

La Filosofía de Python: Duck Typing

"Si camina como un pato y grazna como un pato, entonces es un pato."

A Python no le importa la clase del objeto, solo si tiene el método que intentas llamar.

7 / 12

Polimorfismo sin Herencia

Estas clases no tienen relación entre sí, pero ambas tienen un método `avanzar()`.

class Coche:
    def avanzar(self):
        print("El coche avanza por la carretera.")

class Barco:
    def avanzar(self):
        print("El barco avanza por el agua.")
8 / 12

Duck Typing en Acción

Esta función funciona con cualquier objeto que tenga un método `avanzar()`, sin importar su clase.

def mover_vehiculo(Vehiculo):
    Vehiculo.avanzar()

mi_coche = Coche()
mi_barco = Barco()

mover_vehiculo(mi_coche)
mover_vehiculo(mi_barco)
9 / 12

Polimorfismo con Operadores

El operador + es polimórfico. Se comporta de forma diferente según los tipos de datos.

# Para números, es suma
print(5 + 2) # 7

# Para strings, es concatenación
print("Hola" + " " + "Mundo") # "Hola Mundo"

# Para listas, es unión
print([1, 2] + [3, 4]) # [1, 2, 3, 4]
10 / 12

Beneficios del Polimorfismo

Permite escribir código más flexible y genérico, que puede trabajar con objetos de diferentes tipos sin necesidad de `if/else` para comprobar la clase de cada uno.

11 / 12

Conclusión

El polimorfismo, especialmente a través del Duck Typing, es una de las características que hacen a Python tan flexible y dinámico.
